K primes pattern

I was looking for a way to iterate through prime number k's. Odd-length kmers are desireable because they cannot be reverse complements of themselves [citeme]. Primes are desireable because they cannot be perfect repeats. An ideal iteration pattern would avoid including both twins in twin prime pairs (I have no specific preference about whether the chosen is the larger or the smaller). Re-evaluating data with both twins in a twin pair, where the windows are so similarly sized, seems like it would be for minimal gain. With the same principle of trying to avoid reprocessing data for minimal gain, an ideal iteration would steadily increase in distance between the current k-length and the next. I stumbled upon this pattern, which may be a useful way to limit the analysis space.

julia> step = 2
2

julia> current_prime = 11
11

julia> while Primes.isprime(current_prime)
           current_prime = current_prime + step
           step += 2
           @show current_prime
       end
